How this brief was tailored

A journey shaped to an offline app

The requested journey assumes a networked, account-based product. This app was deliberately designed fully offline with no sign-in, so three items were adapted rather than bolted on — clarity beats checklist-completeness.

Removed

Authentication & Logout

No accounts, no server, nothing to sign into. Data lives on-device. Sharing happens through a file, not a login. Removing these screens removes friction and a privacy surface.

Reframed

Profile → Household

"Profile" becomes a lightweight Household card: family size (default 4), enabled meal slots, and the repeat-alert window. Local preferences, not an identity.

Reframed

Notifications → Reminders

No push. Optional local reminders ("Plan next week", set for Sunday morning) via the on-device scheduler. Off by default; opt-in in Settings.

Design north star. The moment that must feel effortless is opening the app and seeing this week already handled. Every other screen exists to feed that one. Reduce decisions, never add them.

Visual direction

The Airbnb warm-up

Same Material 3 bones, warmer skin. Airbnb's feel isn't a colour — it's warm neutrals, roomy layouts, and calm cards. We keep the curry-leaf green as the primary (it's food-right and we validated it), and warm everything around it. A muted coral appears only as a sparing accent (e.g. the "today" marker), never as the main action. Dish imagery is deferred to a future iteration — v1 cards are clean and text-forward.

Neutrals

Cosy greige

Cool green-grey surfaces → warm off-white (#FAF7F3) and warm greys. Text is a warm near-black (#221F1B), like Airbnb's.

Space

Room to breathe

Bigger cards, larger headings, more air between rows. The UI recedes; the food leads.

Imagery — later

Deferred to v2

Category illustrations & optional dish photos are parked for a future iteration. v1 leads with type on warm surfaces.

Accent

Coral, sparingly

A single warm coral (#E14B5A) for small highlights like the "today" marker. Green still owns primary actions.

Semantic ≠ accent

Turmeric is a signal

Amber never decorates. It means "repeat detected." Keeping it rare makes it legible the instant it appears.

Contrast

AA verified

Every text/background pair meets ≥4.5:1 (body) or ≥3:1 (large / UI). Primary on white = 4.7:1; on-surface = 14:1.

Dynamic colour

Material You ready

These are the brand fallback. On Android 12+, honour the user's wallpaper palette; keep turmeric as a fixed custom role.

Foundations · 03

Spacing — an 8dp grid

All layout snaps to an 8dp rhythm, with a 4dp half-step for tight component internals. Screen margins are 16dp; cards pad 16dp; sections breathe at 24–32dp. Touch targets never drop below 48×48dp.

Buttons

One primary per screen

Filled = the single most important action. Tonal for secondary, outlined/text for tertiary. Never two filled buttons competing.

Chips

Role vs filter

Assist/role chips label an item's role; filter chips (with check) narrow lists. Same shape, different jobs — kept visually distinct by fill.

Feedback

Snackbar, not toast

Destructive-adjacent actions (delete, replace) always offer Undo in a snackbar for ~5s before committing.

Structure

Navigation map & user flow

Four top-level destinations in a bottom navigation bar — reachable one-handed, flat, no nested tabs. Settings sits in the app-bar (rarely visited). Adaptive: the bar becomes a navigation rail on foldables / large screens ≥600dp.

Delight, sparingly

Micro-interactions worth building

Each earns its place by making an action clearer or more satisfying — none are decoration. All respect reduced-motion.

Signature moment

The week fills in

On "Generate", day cards cascade in top-to-bottom (40ms stagger, emphasized-decelerate). It reads as the app doing the work for you.

Swap

Rotating swap icon

The swap_horiz glyph spins 180° and the new dish name cross-fades up — a tiny confirmation the slot changed.

Repeat alert

Amber, then settle

The alert pill fades + rises 4dp once when a repeat appears; it never pulses or nags after.

FAB

Extend on intent

Extended "Generate week" collapses to icon-only on scroll-down, re-extends on scroll-up — content first, action within reach.

Undo

Forgiving deletes

Delete animates the row out but holds the data for a 5s snackbar Undo before committing to the database.

Editor

Container transform

Tapping a combo card / FAB expands it into the editor along a shared container — spatial continuity, no jarring cut.

Platform

Android-specific UX best practices

Design decisions that make this feel native to Android, not a ported iOS app.

  • Predictive back — support Android 14+ predictive back gestures; back always dismisses sheets/dialogs before leaving a screen.
  • Edge-to-edge — draw behind the status & navigation bars; respect insets so content and FAB never hide under the gesture bar.
  • Material You — adopt dynamic colour from wallpaper on Android 12+, with the curry-leaf palette as fallback; turmeric stays fixed as a semantic role.
  • System theme — follow the OS light/dark setting automatically; offer a manual override in Settings.
  • One primary action — a single FAB or filled button per screen, placed in the bottom-right thumb arc.
  • Bottom nav ergonomics — 3–5 flat destinations, labels always visible, active item filled + tonal pill.
  • Adaptive layout — bottom bar → navigation rail at ≥600dp; two-pane list/detail on tablets & unfolded foldables.
  • Storage Access Framework — export/import via the system file picker; no storage permissions needed.
  • Haptics — light tick on generate-complete and destructive confirm; nothing gratuitous.
  • Back-guard — unsaved editor changes prompt before discarding; snackbars, never blocking toasts, for feedback.
  • Touch targets — every interactive element ≥48×48dp with ≥8dp separation.
